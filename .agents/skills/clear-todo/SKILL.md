# 清空待办工作流

当用户说"清空 todo"或"clear todo"时，按以下流程执行。

**核心规则：绝对不可以删除任何 TODO 条目。只能勾选或添加新条目。**

---

## 第一步：读取 TODO.md

读取根目录 `TODO.md`，收集所有未完成的 `- [ ]` 条目。

如果没有任何未完成条目，直接告知用户"没有待办事项"，结束。

---

## 第二步：分类

将待办条目按类别分组，向用户展示分类结果：

```
共 N 个待办，分类如下：

**Bug 修复**
- [ ] xxx

**功能开发**
- [ ] xxx
- [ ] xxx

**重构/优化**
- [ ] xxx

**文档/杂项**
- [ ] xxx
```

分类后询问用户确认，然后开始逐个完成。

---

## 第三步：逐个完成

依次处理每个条目：

1. **宣布**：当前正在处理哪个条目
2. **执行**：完成该任务（代码改动、测试、文档等）
3. **验证**：运行 `pnpm lint` 和 `pnpm test`（如适用）
4. **标记**：将 `TODO.md` 中对应的 `- [ ]` 改为 `- [x]`

每完成一个，简要汇报结果，然后继续下一个。

---

## 第四步：完成汇报

全部处理完毕后，汇总汇报：

```
清空完成，共处理 N 个待办：
- [x] 条目1 — 已完成
- [x] 条目2 — 已完成
- [ ] 条目3 — 未能完成（原因）
```

**如果有条目无法完成**（如需要用户决策、超出能力范围），保留为 `- [ ]` 并说明原因，绝不删除。
